124 /* warning take care not to do something like this:
125 
126  CATCH(errmgr) {
127  cmd1....
128  return 1;
129  } FAIL(errmgr) {
130  cmd2...
131  return 0;
132  }
133 
134  The right way to use it is:
135 
136  {
137  type result;
138 
139  CATCH(errmgr) {
140  cmd1....
141  result = value;
142  } FAIL(errmgr) {
143  cmd2...
144  result = 1;
145  }
146  return(result);
147  }
148  I.e. return inside CATCH/FAIL may cause damage of the stack
